# 近10期命中次数统计说明

## 功能澄清

第二层"近10期次数统计"的逻辑是正确的，其设计目的为：

**以全码类结果为依据，统计在验证大量不同类型公式时，每期有多少组公式命中特码**

## 统计逻辑详解

### 1. 统计范围
- **期数范围**: 最多10期（根据验证期数调整）
- **公式范围**: 所有验证的公式（包括不同类型）
- **数据来源**: 第四层扩展结果（`expandedResults`）

### 2. 统计流程

```
每期统计：
├── 获取该期开奖特码
├── 遍历所有公式
│   ├── 获取该公式在该期的结果
│   ├── 将结果转换为号码
│   └── 统计命中特码的次数
└── 累加所有公式的命中次数
```

### 3. 具体实现

```typescript
// 对每个期数，统计所有公式命中特码的总次数
for (const period of periodsToCount) {
  const actualTeNum = periodData.numbers[6];  // 该期特码
  let hitCount = 0;
  
  for (const result of results) {  // 遍历所有公式
    const periodResult = result.periodResults.find(pr => pr.period === period);
    if (periodResult) {
      // 将该公式的扩展结果转换为号码
      for (const value of periodResult.expandedResults) {
        const numbers = convertResultToNumbers(
          resultToText(value, result.formula.resultType, zodiacYear),
          result.formula.resultType,
          zodiacYear
        );
        // 统计命中特码的次数
        for (const num of numbers) {
          if (num === actualTeNum) {
            hitCount++;
          }
        }
      }
    }
  }
  counts.push(hitCount);
}
```

## 统计意义

### 1. 整体命中率分析
- **横向对比**: 比较不同期数的命中情况
- **趋势识别**: 识别命中率的趋势变化
- **效果评估**: 评估公式组合的整体效果

### 2. 公式质量评估
- **命中集中度**: 某些期数命中特别多说明公式组合好
- **稳定性**: 各期命中次数的差异反映稳定性
- **覆盖度**: 统计所有公式的综合覆盖能力

### 3. 实际应用场景

**示例解释**：
- 第1期: 15次命中 → 15组公式在第1期命中了特码
- 第2期: 8次命中 → 8组公式在第2期命中了特码
- 第3期: 12次命中 → 12组公式在第3期命中了特码

## 与其他统计的区别

### 1. vs 全码类结果统计
- **近10期命中次数**: 统计命中特码的公式组数
- **全码类结果**: 统计各号码在所有公式结果中的出现频次

### 2. vs 按类型统计
- **近10期命中次数**: 跨所有类型的综合命中统计
- **按类型统计**: 按结果类型分组的命中分布

## 显示格式

### 结果展示
```
[近10期开出次数08,12,15,06,09,14,11,07,13,10]
```

**解读**：
- 右边是最新一期
- 每个数字表示该期命中特码的公式组数
- 数值越大说明该期公式组合效果越好

### 预测 vs 回溯模式

**预测模式**：
- 统计最近10期历史数据的命中次数
- 右边显示最新一期的命中数

**回溯模式**：
- 从目标期数开始向后统计
- 右边显示目标期数的命中数

## 注意事项

1. **不是号码统计**: 统计的是命中次数，不是号码分布
2. **包含所有类型**: 不区分公式类型，统计所有公式
3. **基于转换结果**: 使用第四层扩展结果转换后的号码
4. **反映综合效果**: 体现所有公式的整体命中能力

## 总结

第二层"近10期次数统计"是一个重要的综合指标，用于评估所有验证公式的整体命中效果。它通过统计每期有多少组公式命中特码，帮助用户了解公式组合的实战表现和稳定性。
