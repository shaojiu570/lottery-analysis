# 增强公式搜索系统

## 概述

基于原有的搜索系统，我们实现了一个更智能、更高效的公式搜索算法，能够发现更多高质量的新公式，并提供更完善的元素使用频率分析。

## 主要改进

### 1. 增强模式学习系统

#### 新增功能
- **时间衰减分数**: 考虑元素的时效性，最近表现好的元素获得更高权重
- **类型特定分数**: 分析元素在不同结果类型下的表现差异
- **波动性分数**: 评估元素表现的稳定性
- **性能趋势分析**: 识别元素的上升、下降或稳定趋势
- **协同效应分数**: 分析元素间的组合效果

#### 核心接口
```typescript
interface ElementPerformance {
  element: string;
  recentHitRate: number;    // 最近10期命中率
  overallHitRate: number;  // 总体命中率
  trendDirection: 'up' | 'down' | 'stable';
  consistencyScore: number;  // 一致性分数
  synergyScore: number;      // 协同效应分数
}
```

### 2. 智能元素选择器

#### 功能特点
- **多维度评分**: 结合频率、协同效应、趋势、一致性等多个维度
- **动态权重调整**: 根据历史表现动态调整各维度权重
- **回退机制**: 当智能选择不可用时，自动回退到随机选择

#### 评分算法
```typescript
// 基础频率分数 (30%)
score += (10 - freqIndex) * 0.3;

// 协同效应分数 (20%)
score += Math.log(pairFreq + 1) * 0.2;

// 性能趋势分数 (30%)
score += performance.recentHitRate * 0.3;

// 一致性分数 (10%)
score += performance.consistencyScore * 0.1;

// 协同效应分数 (10%)
score += performance.synergyScore * 0.1;
```

### 3. 结构模式挖掘

#### 新增功能
- **公式结构提取**: 将具体元素抽象为类型标识符
- **成功结构识别**: 学习历史上成功的公式结构模式
- **最优配置发现**: 分析不同期数配置下的表现

#### 结构映射规则
```typescript
// 数字元素 -> NUM
'期数|总分|平[1-6]号|特号' → 'NUM'

// 头数元素 -> HEAD  
'平[1-6]头|特头' → 'HEAD'

// 尾数元素 -> TAIL
'平[1-6]尾|特尾' → 'TAIL'

// 其他类型...
```

### 4. 四阶段搜索策略

#### 阶段1: 模式驱动搜索
- 基于历史成功模式生成候选公式
- 优先使用高频、高协同效应的元素组合
- 利用学习的结构模式指导搜索方向

#### 阶段2: 随机多样性搜索
- 补充随机性，避免陷入局部最优
- 70%概率使用模式学习结果，30%纯随机
- 确保搜索空间的充分探索

#### 阶段3: 全元素深度探索
- 确保所有元素都有被使用的机会
- 使用智能元素选择器提高效率
- 针对不常用元素进行重点探索

#### 阶段4: 跨类型复用
- 将优质公式中的元素组应用到其他结果类型
- 尝试不同的规则(D/L)组合
- 最大化优质元素组合的价值

## 元素频率分析改进

### 1. 多维度分析

#### 时间维度
- **短期表现**: 最近10期的命中率
- **长期表现**: 整体历史命中率
- **趋势分析**: 上升/下降/稳定趋势

#### 稳定性维度
- **一致性分数**: 命中率方差分析
- **波动性评估**: 表现的波动程度

#### 关联性维度
- **协同效应**: 与其他元素的组合效果
- **最佳配对**: 历史上表现最好的元素对

### 2. 动态权重系统

根据元素的实时表现动态调整其在搜索中的权重：
```typescript
const dynamicWeight = baseFrequency * 
                    timeDecayFactor * 
                    trendBoost * 
                    consistencyBonus * 
                    synergyMultiplier;
```

### 3. 缓存优化

- **元素性能缓存**: 避免重复计算元素性能指标
- **模式学习缓存**: 存储学习到的公式模式
- **成功公式缓存**: 记录历史上成功的公式

## 使用效果

### 1. 搜索效率提升
- **智能剪枝**: 减少低效公式的生成和验证
- **目标导向**: 基于历史成功模式指导搜索
- **并行优化**: 多阶段并行搜索提高效率

### 2. 结果质量改善
- **更高命中率**: 发现更多符合目标命中率的公式
- **更好多样性**: 避免公式类型过于集中
- **更强实用性**: 优先选择稳定可靠的元素组合

### 3. 分析能力增强
- **深度洞察**: 提供元素的多维度性能分析
- **趋势预测**: 识别元素的发展趋势
- **优化建议**: 基于分析结果提供优化建议

## 配置建议

### 快速搜索 (Fast)
- 适用于快速验证和初步探索
- 重点使用模式驱动搜索
- 限制随机和深度探索的时间

### 标准搜索 (Standard)  
- 平衡效率和质量的最佳选择
- 完整执行四个搜索阶段
- 适合大多数使用场景

### 深度搜索 (Deep)
- 最大化发现新公式的概率
- 增加深度探索的迭代次数
- 适合重要决策前的全面分析

增强的搜索系统通过多维度分析、智能选择和结构化搜索，显著提升了发现高质量新公式的能力。系统不仅能够找到更多符合要求的公式，还能提供深入的元素性能分析，帮助用户更好地理解和优化公式策略。
